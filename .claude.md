# Crabigator Stats - Development Guidelines

## Pre-Commit Requirements

Before pushing any commits, ensure the following checks pass:

### 1. Linter (ESLint)
Run ESLint to check for code quality issues:
```bash
npm run lint
```

**All linting errors must be fixed** before committing. The project uses:
- Next.js recommended rules (`eslint-config-next/core-web-vitals`)
- TypeScript ESLint rules (`eslint-config-next/typescript`)
- Additional React and code quality rules

Common linting commands:
- `npm run lint` - Check for linting errors
- `npm run lint -- --fix` - Auto-fix linting errors where possible

### 2. TypeScript Type Checking
Run TypeScript compiler to check for type errors:
```bash
npx tsc --noEmit
```

**All TypeScript errors must be resolved** before committing.

### 3. Build Check
Ensure the project builds successfully:
```bash
npm run build
```

**The build must complete without errors** before committing.

## Workflow

1. Make your code changes
2. Run `npm run lint -- --fix` to auto-fix linting issues
3. Manually fix any remaining linting errors
4. Run `npx tsc --noEmit` to check for TypeScript errors
5. Fix all TypeScript errors
6. Run `npm run build` to ensure the project builds
7. Commit your changes
8. Push to the repository

## Code Quality Standards

- Use TypeScript for type safety
- Follow React and Next.js best practices
- Avoid using `any` type (use `unknown` or specific types)
- Prefix unused variables with underscore (`_variableName`)
- Use `const` over `let` when variables aren't reassigned
- Never use `var`
- Console statements should only be `console.warn()` or `console.error()`
